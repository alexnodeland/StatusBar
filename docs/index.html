StatusBar

A native macOS menu bar app that monitors all your status pages in one place. Supports Atlassian Statuspage, incident.io, and Instatus with automatic provider detection. No Dock icon. No Electron. Just a single icon that tells you everything.

Download View on GitHub

Features

See it in action

All Your Services, One Icon

Click the menu bar icon to see every service at once. Color-coded indicators show what’s healthy and what needs attention. A badge on the icon tells you how many sources have issues — without opening the popover.

Drill Into the Details

Tap any source to see individual components and active incidents. Expand an incident to follow the full timeline — from investigation through resolution — without leaving the menu bar.

Native macOS Integration

Lives in the menu bar as a checkmark when all clear, or a warning with a badge count when issues arise. Enable notifications to get native macOS alerts the moment a service status changes.

Automate & Integrate

Connect StatusBar to your workflow

Webhooks

Push status change alerts to Slack, Discord, Microsoft Teams, or any HTTP endpoint. Configure multiple webhooks in Settings → Webhooks.

Slack Discord Teams Generic JSON
PlatformPayload shape
Slack{ "text": "..." }
Discord{ "content": "..." }
Teams{ "@type": "MessageCard", ... }
Generic{ "source", "title", "body" }

Webhooks fire on every status change. Each webhook has an enable/disable toggle and a 10-second request timeout.

Script Hooks

Drop executable scripts into the hooks folder and they run automatically on status events. Trigger CI pipelines, write to logs, post to APIs—anything you can script.

1 Open Settings → Hooks → Add Example Hook, or create your own script
2 Make it executable: chmod +x my-hook
3 It runs on every event—check $STATUSBAR_EVENT to filter 
#!/bin/bash
# Log status changes to a file
[ "$STATUSBAR_EVENT" = "on-status-change" ] || exit 0

LOG="$HOME/Library/Logs/StatusBar/hooks.log"
mkdir -p "$(dirname "$LOG")"
echo "[$(date)] $STATUSBAR_SOURCE_NAME: $STATUSBAR_TITLE" >> "$LOG"
EventWhen
on-status-changeSource status severity changes
on-refreshAll sources finish refreshing
on-source-addA new source is added
on-source-removeA source is removed

STATUSBAR_EVENTAll

STATUSBAR_WORST_LEVELrefresh

A full JSON payload is also piped to stdin. Scripts can be any language—just add a shebang and make them executable. 30-second timeout per execution.

URL Scheme

Control the app from Terminal, browsers, Raycast, Alfred, or macOS Shortcuts using statusbar:// deep links.

open "statusbar://open"
Show the popover 
open "statusbar://open?source=GitHub"
Navigate to a source 
open "statusbar://refresh"
Refresh all sources 
open "statusbar://add?url=https://status.openai.com&name=OpenAI"
Add a new source 
open "statusbar://remove?name=GitHub"
Remove by name 
open "statusbar://settings?tab=webhooks"
Jump to a settings tab

AppleScript

Full Cocoa Scripting dictionary. Query status, refresh, add and remove sources from Script Editor, JXA, osascript, or any OSA-compatible tool.

Script Editor JXA osascript Shortcuts 
osascript -e 'tell application "StatusBar" to get name of every source'
osascript -e 'tell application "StatusBar" to get worst status'
osascript -e 'tell application "StatusBar" to refresh'

Open Script Editor → File → Open Dictionary → StatusBar to browse the full scripting dictionary.

tell application "StatusBar"
    -- Read properties
    get name of every source
    get status of source "GitHub"
    get worst status
    get issue count

    -- Actions
    refresh
    add source "https://status.openai.com" named "OpenAI"
    remove source "OpenAI"
end tell

Get Started

Up and running in seconds

1 Download StatusBar-vX.Y.Z-universal.zip from the Releases page
2 Extract the ZIP
3 Drag StatusBar.app to /Applications
4 On first launch, right-click → Open to bypass Gatekeeper
Note: The app is ad-hoc signed, not notarized. The universal binary runs natively on both Apple Silicon and Intel Macs. 
chmod +x build.sh
./build.sh
open ./build/StatusBar.app

Or compile directly:

swiftc StatusBarApp.swift -parse-as-library \
  -o StatusBar -framework SwiftUI -framework AppKit \
  -target arm64-apple-macosx14.0
./StatusBar

Adding Sources

Open Settings and click + to add any status page by name and URL. The app auto-detects the provider — Atlassian Statuspage, incident.io, or Instatus. You can also import a JSON file to add many at once.

Select services below, export as JSON, then import the file in StatusBar via Settings → Data → Import Sources.
